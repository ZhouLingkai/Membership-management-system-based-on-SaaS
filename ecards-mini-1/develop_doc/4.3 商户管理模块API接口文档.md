# 会员管理系统 API 接口文档

## 1. 统一返回格式

所有接口均使用 "Result" 包装器返回数据，结构如下：

| 字段名     | 类型     | 描述                          |
| ------- | ------ | --------------------------- |
| code    | Int    | 状态码：200 成功，非 200 失败（含业务错误码） |
| data    | Object | 响应数据对象（成功时非空，失败时可空）         |
| message | String | 响应消息：成功时返回操作结果，失败时返回具体错误提示  |

#### 4.3.2 接口 1：商户注册【已修改】

* 业务背景：普通用户（USER 角色）申请升级为商户（MERCHANT 角色），系统提供两种通道：
  * **免认证通道（applicationType=1）**：只需填写商家名称和二级密码，立即开通7天测试期（certification=2），测试期内需补充资质转正式审核
  * **直接认证通道（applicationType=2）**：提交完整资质信息，系统自动生成商家名称（"商家"+UUID后5位），二级密码默认等于用户密码，进入审核流程（certification=3）

* 认证：需携带用户普通令牌；校验令牌有效性、用户角色（仅 USER 可申请）；校验 Device-Id 与令牌绑定设备一致。已经是商户的用户不能再次申请。

* URL：/api/v1/merchants/registration

* HTTP Method：POST

* 请求头：

| 请求头字段 | 是否必填 | 示例值                     | 描述               |
|------------|----------|----------------------------|--------------------|
| Content-Type | 是      | application/json           | 请求体数据格式     |
| Device-Id   | 是      | device\_1234567890abcdef    | 设备唯一标识       |
| Authorization | 是    | Bearer eyJhbGciOiJSUzI1Ni...| 用户普通令牌       |

* 请求参数：

| 参数名         | 类型    | 是否必填 | 示例值                     | 描述                                      |
|----------------|---------|----------|----------------------------|-------------------------------------------|
| userId          | String  | 是       | uuid-1234-5678-90ab-cdef   | 申请人用户 ID（与令牌中 user\_id 一致）        |
| applicationType | Integer | 是       | 1                          | 申请方式：1-免认证通道，2-直接认证通道           |
| merchantName    | String  | 条件必填 | XX 连锁便利店               | 商户名称（applicationType=1时必填，长度 1-100 位） |
| sndPswd         | String  | 条件必填 | password123                | 二级密码（applicationType=1时必填，明文，长度≥8位） |
| numStores       | Integer | 条件必填 | 3                          | 店铺规模（applicationType=2时必填）            |
| numMembers      | String  | 条件必填 | "1000-2000人"                   | 会员规模（applicationType=2时必填）            |
| storeName       | String  | 条件必填 | XX便利店第一分店            | 第一家店铺名称（applicationType=2时必填）       |
| storePhotos     | String  | 条件必填 | base64EncodedPhoto         | 门头店照（applicationType=2时必填，Base64编码） |
| businessLicense | String  | 条件必填 | base64EncodedLicenseImg    | 营业执照图片（applicationType=2时必填，Base64） |

* 响应结果：

| 字段名           | 类型    | 示例值                     | 描述                                      |
|------------------|---------|----------------------------|-------------------------------------------|
| code             | Int     | 200                        | 状态码：200 成功，403 - 角色不允许 / 参数不完整 |
| data.merchantId  | String  | uuid-5678-90ab-cdef-1234   | 商户 ID（UUID 字符串）                     |
| data.merchantName| String  | XX 连锁便利店               | 商户名称                                  |
| data.certification| Integer| 2                          | 认证状态：2-测试中，3-审核中                |
| data.applyTime   | String  | 2025-10-27 09:30:00        | 申请时间（yyyy-MM-dd HH:mm:ss 格式）       |
| data.testExpireTime| String| 2025-11-03 09:30:00        | 测试期过期时间（仅 certification=2 时返回） |
| message          | String  | 商户申请成功，已开通7天测试期 | 响应消息（失败时返回具体错误信息）        |

* 响应示例（免认证通道）：



```json
{
  "code": 200,
  "data": {
    "merchantId": "uuid-5678-90ab-cdef-1234",
    "merchantName": "XX连锁便利店",
    "certification": 2,
    "applyTime": "2025-10-27 09:30:00",
    "testExpireTime": "2025-11-03 09:30:00"
  },
  "message": "商户申请成功，已开通7天测试期，请及时在测试期内补充资质认证"
}
```

* 响应示例（直接认证通道）：



```json
{
  "code": 200,
  "data": {
    "merchantId": "uuid-5678-90ab-cdef-1234",
    "merchantName": "商家-3d4f1",
    "certification": 3,
    "applyTime": "2025-10-27 09:30:00"
  },
  "message": "商户资质提交成功，待审核（预计1-2个工作日）"
}
```

* 可能的错误码：

| 错误码 | 说明 |
|--------|------|
| 403 | 用户ID与令牌不匹配 |
| 20003 | 用户不存在 |
| 20005 | 权限不足（该用户已是商户/已申请过商户/有待审核记录）|
| 90001 | 参数错误（申请方式只能是1或2/缺少必填参数）|

#### 4.3.3 接口 2：商户基础信息查询



* 业务背景：商户查询自身基础信息，包括商户名称、联系信息、资质状态、测试期倒计时等，不返回敏感信息。

* 认证：需携带商户普通令牌；校验令牌有效性、用户角色（仅 MERCHANT）；校验令牌中 merchant\_id 与查询商户 ID 一致

* URL：/api/v1/merchants/info

* HTTP Method：GET

* 请求头：同 "商户注册" 请求头

* 请求参数（Query 参数）：

| 参数名         | 类型    | 是否必填 | 示例值                     | 描述                                      |
|----------------|---------|----------|----------------------------|-------------------------------------------|
| merchantId      | String  | 是       | uuid-5678-90ab-cdef-1234   | 商户 ID（与令牌中 merchant\_id 一致）          |

* 响应结果：

| 字段名               | 类型    | 示例值                     | 描述                                      |
|----------------------|---------|----------------------------|-------------------------------------------|
| code                 | Int     | 200                        | 状态码：200 成功，404 - 商户不存在            |
| data.merchantId      | String  | uuid-5678-90ab-cdef-1234   | 商户 ID                                    |
| data.merchantName    | String  | XX 连锁便利店               | 商户名称                                  |
| data.certification   | Integer | 1                          | 认证状态：1-已认证，2-测试中，3-审核中，4-审核拒绝，5-测试期过 |
| data.testExpireTime  | String  | 2025-11-03 09:30:00        | 测试期过期时间（certification=2时返回）    |
| data.remainingDays   | Integer | 5                          | 测试期剩余天数（certification=2时返回）    |
| data.contactPhone    | String  | 1386931234                | 联系电话          |
| data.contactEmail    | String  | xx\*\*@[xxx.com](https://xxx.com)               | 联系邮箱            |

| data.storeCount      | Int     | 3                          | 关联店铺数量                              |
| message              | String  | 信息查询成功               | 响应消息（失败时返回具体错误信息）        |

* 响应示例（已认证商户）：



```json

    "certification": 1,

    "createTime": "2025-10-27 14:00:00",
    "storeCount": 3
  },
  "message": "信息查询成功"

```

{

    "merchantName": "XX连锁便利店",

    "remainingDays": 5,

}

| 错误码 | 说明 |
|--------|------|
| 90001 | 参数错误（商户不存在）|
| 20003 | 用户不存在 |

#### 4.3.4 接口 3：商户基础信息修改



* 业务背景：已审核通过的商户修改非敏感基础信息（联系电话、邮箱、商户简介），敏感信息（商户名称、营业执照）需单独走资质更新流程。

* 认证：需携带商户普通令牌；校验令牌有效性、用户角色（仅 MERCHANT）；校验 merchant\_id 一致性

* URL：/api/v1/merchants/info

* HTTP Method：PUT

* 请求头：同 "商户注册" 请求头

* 请求参数：

| 参数名         | 类型    | 是否必填 | 示例值                     | 描述                                      |
|----------------|---------|----------|----------------------------|-------------------------------------------|
| merchantId      | String  | 是       | uuid-5678-90ab-cdef-1234   | 商户 ID（与令牌一致）                      |
| contactEmail    | String  | 否       | aesEncryptedNewEmail       | 新联系邮箱（不加密，不填则不修改）  |
| merchantIntro   | String  | 否       | 提供 24 小时便利服务         | 商户简介（长度 0-500 位，不填则不修改）      |

* 响应结果：

| 字段名           | 类型    | 示例值                     | 描述                                      |
|------------------|---------|----------------------------|-------------------------------------------|
| code             | Int     | 200                        | 状态码：200 成功，403 - 权限不足              |
| data.updateTime  | String  | 2025-10-27 16:30:00        | 信息更新时间（yyyy-MM-dd HH:mm:ss 格式）   |
| message          | String  | 商户信息修改成功           | 响应消息（失败时返回具体错误信息）        |

* 响应示例：



```json
{
  "code": 200,
  "data": {
    "updateTime": "2025-10-27 16:30:00"
  },
  "message": "商户信息修改成功"
}
```

* 可能的错误码：

| 错误码 | 说明 |
|--------|------|
| 90001 | 参数错误（商户不存在/没有需要更新的字段）|
| 20005 | 权限不足（当前商户状态不允许修改信息）|

#### 4.3.5 接口 4：商户二级密码修改

* 业务背景：商户修改二级密码（用于获取管理令牌执行危险操作），需验证原二级密码有效性，新密码需符合复杂度要求（长度≥8位）。

* 认证：需携带商户普通令牌；校验令牌有效性、用户角色（仅 MERCHANT）

* URL：/api/v1/merchants/snd-pwd

* HTTP Method：PUT

* 请求头：同 "商户注册" 请求头

* 请求参数：

| 参数名         | 类型    | 是否必填 | 示例值                     | 描述                                      |
|----------------|---------|----------|----------------------------|-------------------------------------------|
| merchantId      | String  | 是       | uuid-5678-90ab-cdef-1234   | 商户 ID（与令牌一致）                      |
| oldSndPswd      | String  | 是       | oldPassword123             | 原二级密码（明文）                         |
| newSndPswd      | String  | 是       | newPassword456             | 新二级密码（明文，长度≥8位）               |
| confirmSndPswd  | String  | 是       | newPassword456             | 确认新二级密码（需与 newSndPswd 一致）      |

* 响应结果：

| 字段名           | 类型    | 示例值                     | 描述                                      |
|------------------|---------|----------------------------|-------------------------------------------|
| code             | Int     | 200                        | 状态码：200 成功，400 - 原密码错误 / 两次密码不一致 |
| data.updateTime  | String  | 2025-10-27 15:00:00        | 二级密码更新时间                          |
| message          | String  | 二级密码修改成功，请重新获取管理令牌 | 响应消息（失败时返回具体错误信息）        |

* 响应示例：



```json
{
  "code": 200,
  "data": {
    "updateTime": "2025-10-27 15:00:00"
  },
  "message": "二级密码修改成功，请重新获取管理令牌"
}
```

* 可能的错误码：

| 错误码 | 说明 |
|--------|------|
| 90001 | 参数错误（商户不存在/两次密码不一致/原密码错误）|
| 20005 | 权限不足（当前商户状态不允许修改） |

#### 4.3.6 接口 5：商户二级密码重置（通过验证码）【新增】

* 业务背景：商户忘记二级密码时，通过手机验证码重置二级密码，无需验证原密码。

* 认证：无需令牌（通过手机验证码认证）

* URL：/api/v1/merchants/snd-pwd/reset

* HTTP Method：POST

* 请求头：

| 请求头字段 | 是否必填 | 示例值                     | 描述               |
|------------|----------|----------------------------|--------------------|
| Content-Type | 是      | application/json           | 请求体数据格式     |
| Device-Id   | 是      | device\_1234567890abcdef    | 设备唯一标识       |

* 请求参数：

| 参数名         | 类型    | 是否必填 | 示例值                     | 描述                                      |
|----------------|---------|----------|----------------------------|-------------------------------------------|
| merchantId      | String  | 是       | uuid-5678-90ab-cdef-1234   | 商户 ID                                   |
| phone           | String  | 是       | AES加密后的手机号           | 手机号（AES加密）                          |
| verifyCode      | String  | 是       | 123456                     | 短信验证码                                |
| newSndPswd      | String  | 是       | newPassword456             | 新二级密码（明文，长度≥8位）               |
| platform        | String  | 是       | WEB                        | 平台标识（WEB/ANDROID/IOS/WECHAT）        |

* 响应结果：

| 字段名           | 类型    | 示例值                     | 描述                                      |
|------------------|---------|----------------------------|-------------------------------------------|
| code             | Int     | 200                        | 状态码：200 成功                           |
| data.resetTime   | String  | 2025-10-27 15:00:00        | 二级密码重置时间                           |
| message          | String  | 二级密码重置成功           | 响应消息                                  |

* 响应示例：

```json
{
  "code": 200,
  "data": {
    "resetTime": "2025-10-27 15:00:00"
  },
  "message": "二级密码重置成功"
}
```

* 可能的错误码：

| 错误码 | 说明 |
|--------|------|
| 90001 | 参数错误（商户不存在/手机号格式错误）|
| 20002 | 验证码无效或已过期 |
| 20003 | 关联用户不存在 |
| 20005 | 权限不足（手机号与商户注册手机不一致/商户状态不允许重置）|

#### 4.3.7 接口 6：商户资质上传/更新【未实现】

> ⚠️ **注意：此接口尚未实现，仅保留设计文档供后续开发参考。**

* 业务背景：商户补充或更新资质信息（如营业执照到期更换、法人信息变更），提交后触发二次审核，审核期间不影响现有店铺运营。

* 认证：需携带商户管理令牌（因涉及敏感信息，需二级密码验证）；校验令牌有效性、角色（仅 MERCHANT）

* URL：/api/v1/merchants/qualification

* HTTP Method：PUT

* 请求头：

| 请求头字段 | 是否必填 | 示例值                     | 描述               |
|------------|----------|----------------------------|--------------------|
| Content-Type | 是      | application/json           | 请求体数据格式     |
| Device-Id   | 是      | device\_1234567890abcdef    | 设备唯一标识       |
| Authorization | 是    | Bearer eyJhbGciOiJSUzI1Ni...| 商户管理令牌       |

* 请求参数：

| 参数名         | 类型    | 是否必填 | 示例值                     | 描述                                      |
|----------------|---------|----------|----------------------------|-------------------------------------------|
| merchantId      | String  | 是       | uuid-5678-90ab-cdef-1234   | 商户 ID（与令牌一致）                      |
| businessLicense | String  | 否       | merchant/xxx/license.jpg   | 新营业执照图片（OSS路径，不填则不更新）    |
| licenseNo       | String  | 否       | aesEncryptedNewLicenseNo   | 新营业执照编号（AES 加密，不填则不更新）    |
| legalPerson     | String  | 否       | aesEncryptedNewLegalName   | 新法人姓名（AES 加密，不填则不更新）        |
| legalIdCard     | String  | 否       | aesEncryptedNewLegalId     | 新法人身份证号（AES 加密，不填则不更新）    |
| updateReason    | String  | 是       | 营业执照到期更换           | 更新原因（长度 10-200 位，必填）            |

* 响应结果：

| 字段名           | 类型    | 示例值                     | 描述                                      |
|------------------|---------|----------------------------|-------------------------------------------|
| code             | Int     | 200                        | 状态码：200 成功，403 - 管理令牌无效          |
| data.auditStatus | String  | WAIT                       | 新资质审核状态：WAIT - 待审核             |
| data.submitTime  | String  | 2025-10-27 11:00:00        | 资质提交时间                              |
| message          | String  | 资质更新提交成功，待审核   | 响应消息                                  |

* 响应示例：

```json
{
  "code": 200,
  "data": {
    "auditStatus": "WAIT",
    "submitTime": "2025-10-27 11:00:00"
  },
  "message": "资质更新提交成功，待审核（预计1个工作日）"
}
```

* 可能的错误码：

| 错误码 | 说明 |
|--------|------|
| 90001 | 参数错误（商户不存在/更新原因过短）|
| 10004 | 令牌权限不足（需要管理令牌）|
| 20005 | 权限不足（当前商户状态不允许更新资质）|

#### 4.3.8 接口 7：商户资质审核结果查询【已修改】

* 业务背景：用户查询自己的商户资质审核结果（包括审核意见），若审核驳回可根据意见修改后重新提交。

* 认证：需携带用户普通令牌；校验令牌有效性；校验 userId 与令牌一致

* URL：/api/v1/merchants/qual-audit

* HTTP Method：GET

* 请求头：同 "商户注册" 请求头

* 请求参数（Query 参数）：

| 参数名         | 类型    | 是否必填 | 示例值                     | 描述                                      |
|----------------|---------|----------|----------------------------|-------------------------------------------|
| userId          | String  | 是       | uuid-1234-5678-90ab-cdef   | 用户 ID（与令牌中 user_id 一致）           |

* 响应结果：

| 字段名               | 类型    | 示例值                     | 描述                                      |
|----------------------|---------|----------------------------|-------------------------------------------|
| code                 | Int     | 200                        | 状态码：200 成功，404 - 无审核记录         |
| data.auditId         | Long    | 1001                       | 审核记录 ID（自增主键）                    |
| data.auditStatus     | String  | WAIT                       | 审核状态：WAIT-待审核，PASSED-通过，REJECTED-驳回 |
| data.submitTime      | String  | 2025-10-27 11:00:00        | 资质提交时间                              |
| data.auditTime       | String  | 2025-10-27 15:30:00        | 审核完成时间（待审核时不返回）             |
| data.rejectReason    | String  | 营业执照模糊，请重新上传   | 驳回原因（仅 REJECTED 时返回）             |
| message              | String  | 审核结果查询成功           | 响应消息                                  |

* 响应示例（待审核）：

```json
{
  "code": 200,
  "data": {
    "auditId": 1001,
    "auditStatus": "WAIT",
    "submitTime": "2025-10-27 11:00:00"
  },
  "message": "审核结果查询成功"
}
```

* 响应示例（审核通过）：

```json
{
  "code": 200,
  "data": {
    "auditId": 1001,
    "auditStatus": "PASSED",
    "submitTime": "2025-10-27 11:00:00",
    "auditTime": "2025-10-27 15:30:00"
  },
  "message": "审核结果查询成功"
}
```

* 响应示例（审核驳回）：

```json
{
  "code": 200,
  "data": {
    "auditId": 1001,
    "auditStatus": "REJECTED",
    "submitTime": "2025-10-27 11:00:00",
    "auditTime": "2025-10-27 15:30:00",
    "rejectReason": "营业执照模糊，请重新上传"
  },
  "message": "审核结果查询成功"
}
```

* 可能的错误码：

| 错误码 | 说明 |
|--------|------|
| 403 | 用户ID与令牌不匹配 |
| 404 | 您还没有提交申请成为商家 |
| 20003 | 用户信息不存在 |
| 20005 | 权限不足（员工账号不能申请成为商家）|

#### 4.3.9 接口 8：商户资质补充提交【已修改】

* 业务背景：商户补充完整资质信息，提交后转为审核状态，等待平台审核。支持多种商户状态提交。

* 认证：需携带商户普通令牌；校验令牌有效性、用户角色（仅 MERCHANT）

* URL：/api/v1/merchants/qualification/submit

* HTTP Method：POST

* 请求头：同 "商户注册" 请求头

* 请求参数：

| 参数名         | 类型    | 是否必填 | 示例值                     | 描述                                      |
|----------------|---------|----------|----------------------------|-------------------------------------------|
| merchantId      | String  | 是       | uuid-5678-90ab-cdef-1234   | 商户 ID（与令牌一致）                      |
| numStores       | Integer | 是       | 3                          | 店铺规模（预计店铺数量）                   |
| numMembers      | String  | 是       | 500-1000                   | 会员规模（预计会员数量范围）               |
| storeName       | String  | 是       | XX便利店第一分店            | 第一家店铺名称                            |
| storePhotos     | String  | 是       | merchant/xxx/storePhoto.jpg | 门头店照（OSS Object路径）                 |
| businessLicense | String  | 是       | merchant/xxx/license.jpg   | 营业执照图片（OSS Object路径）             |

* 认证状态转换规则：

| 原状态（certification） | 转换后状态 | 说明 |
|------------------------|-----------|------|
| 2 - 未认证测试中 | 3 - 审核中 | 7天测试期内提交 |
| 4 - 审核拒绝 | 3 - 审核中 | 重新提交资质 |
| 5 - 未认证测试期过 | 13 - 过期后审核中 | 过期后提交 |
| 14 - 过期后审核拒绝 | 13 - 过期后审核中 | 过期后重新提交 |

* 响应结果：

| 字段名           | 类型    | 示例值                     | 描述                                      |
|------------------|---------|----------------------------|-------------------------------------------|
| code             | Int     | 200                        | 状态码：200 成功                           |
| data.auditStatus | String  | WAIT                       | 审核状态：WAIT - 待审核                    |
| data.submitTime  | String  | 2025-10-29 10:30:00        | 资质提交时间                              |
| data.certification| Integer| 3 或 13                    | 更新后的认证状态                          |
| message          | String  | 资质补充成功，已转入审核流程 | 响应消息                                  |

* 响应示例：

```json
{
  "code": 200,
  "data": {
    "auditStatus": "WAIT",
    "submitTime": "2025-10-29 10:30:00",
    "certification": 3
  },
  "message": "资质补充成功，已转入审核流程（预计1-2个工作日）"
}
```

* 可能的错误码：

| 错误码 | 说明 |
|--------|------|
| 90001 | 参数错误（商户不存在/参数缺失）|
| 20005 | 权限不足（当前商户状态不允许提交资质）|

| 状态值 | 状态名称 | 说明 | 可执行操作 |
|-------|---------|------|-----------|
| 1 | 已认证 | 审核通过的正式商户 | 全部功能可用 |

| 6 | 认证存疑 | 预留状态（暂未使用） | - |
| 7 | 封禁中 | 商家账号被封禁 | 不可进行商家有关操作 |
| 13 | 过期后审核中 | 测试期过后提交资质审核中 | 等待审核 |
| 14 | 过期后审核拒绝 | 过期后审核未通过 | 可重新提交 |
