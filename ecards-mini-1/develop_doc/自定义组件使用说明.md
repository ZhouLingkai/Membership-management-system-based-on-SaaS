# 自定义组件使用说明

本文档介绍项目中两个核心自定义组件的使用方法，包括组件属性、方法、事件以及实际应用示例。

---

## 1. custom-toast（自定义通知组件）

### 1.1 组件简介

自定义消息提示组件，支持成功/危险两种类型，提供滑动和淡入两种动画效果。相比微信原生 `wx.showToast`，该组件支持更长的文本内容和更丰富的视觉效果。

**已应用页面：** `/pages/login/index`

---

### 1.2 快速上手

#### 步骤1：在页面 JSON 中注册组件

```json
{
  "usingComponents": {
    "custom-toast": "../../components/custom-toast/custom-toast"
  }
}
```

#### 步骤2：在页面 WXML 中添加组件

```xml
<custom-toast 
  id="customToast" 
  message="{{toastMessage}}" 
  type="{{toastType}}" 
  animation-type="{{toastAnimationType}}" 
  show="{{showToast}}" />
```

#### 步骤3：在页面 JS 中初始化数据

```javascript
Page({
  data: {
    showToast: false,
    toastMessage: '',
    toastType: 'success',
    toastAnimationType: 'slide'
  }
})
```

#### 步骤4：调用组件显示消息

**方式一：通过组件方法调用（推荐）**

```javascript
// 在页面 JS 中定义封装方法
showCustomToast(message, type = 'success', animationType = 'slide', duration = 2000) {
  const toast = this.selectComponent('#customToast');
  if (toast) {
    toast.showToast(message, type, animationType, duration);
  }
}

// 使用示例
this.showCustomToast('登录成功', 'success');
this.showCustomToast('密码错误', 'danger');
```

**方式二：通过 data 属性控制**

```javascript
this.setData({
  showToast: true,
  toastMessage: '注册成功',
  toastType: 'success',
  toastAnimationType: 'fade'
});
```

---

### 1.3 组件属性

| 属性名 | 类型 | 默认值 | 必填 | 说明 |
|--------|------|--------|------|------|
| message | String | '' | 否 | 消息内容 |
| type | String | 'success' | 否 | 消息类型：`success`（绿色）或 `danger`（红色） |
| animationType | String | 'slide' | 否 | 动画类型：`slide`（顶部滑下）或 `fade`（中间淡入） |
| show | Boolean | false | 否 | 是否显示消息 |
| duration | Number | 2000 | 否 | 显示时长（毫秒） |

---

### 1.4 组件方法

#### showToast()

显示消息提示。

**参数：**
- `message` (String): 消息内容
- `type` (String): 消息类型，可选 `'success'` 或 `'danger'`，默认 `'success'`
- `animationType` (String): 动画类型，可选 `'slide'` 或 `'fade'`，默认 `'slide'`
- `duration` (Number): 显示时长（毫秒），默认 `2000`

**示例：**
```javascript
const toast = this.selectComponent('#customToast');
toast.showToast('操作成功', 'success', 'slide', 2000);
```

#### hideToast()

立即隐藏消息提示。

**示例：**
```javascript
const toast = this.selectComponent('#customToast');
toast.hideToast();
```

---

### 1.5 实际应用案例

以下是登录页面的完整应用示例：

**index.json**
```json
{
  "usingComponents": {
    "custom-toast": "../../components/custom-toast/custom-toast"
  }
}
```

**index.wxml**
```xml
<view class="login-container">
  <!-- 其他内容 -->
  
  <!-- 自定义消息组件 -->
  <custom-toast 
    id="customToast" 
    message="{{toastMessage}}" 
    type="{{toastType}}" 
    animation-type="{{toastAnimationType}}" 
    show="{{showToast}}" />
</view>
```

**index.js**
```javascript
Page({
  data: {
    showToast: false,
    toastMessage: '',
    toastType: 'success',
    toastAnimationType: 'slide'
  },

  // 封装消息提示方法
  showCustomToast(message, type = 'success', animationType = 'slide', duration = 2000) {
    const toast = this.selectComponent('#customToast');
    if (toast) {
      toast.showToast(message, type, animationType, duration);
    }
  },

  // 登录处理示例
  async handleLogin() {
    try {
      const res = await post('/v1/users/login', headers, data);
      
      if (res.code === 200) {
        this.showCustomToast('登录成功', 'success');
        // 跳转逻辑...
      } else {
        this.showCustomToast(res.message, 'danger');
      }
    } catch (error) {
      this.showCustomToast('登录失败，请稍后重试', 'danger');
    }
  }
})
```

## 3. step-form（步骤表单组件）

### 3.1 组件简介

多步骤表单组件，提供可视化进度指示、步骤跳转、滑动切换等功能。适用于需要分步收集信息的场景，如注册、升级、问卷调查等。

**已应用页面：** `/pages/upgrade-merchant/upgrade-merchant`

---

### 3.2 快速上手

#### 步骤1：在页面 JSON 中注册组件

```json
{
  "usingComponents": {
    "step-form": "/components/step-form/step-form"
  }
}
```

#### 步骤2：在页面 WXML 中添加组件

```xml
<step-form 
  id="stepForm"
  steps="{{steps}}"
  currentStep="{{currentStep}}"
  bindstepchange="onStepChange">
  
  <!-- 步骤1内容 -->
  <view class="step-panel" wx:if="{{currentStep === 0}}">
    <text>第一步的表单内容</text>
  </view>
  
  <!-- 步骤2内容 -->
  <view class="step-panel" wx:if="{{currentStep === 1}}">
    <text>第二步的表单内容</text>
  </view>
  
  <!-- 更多步骤... -->
</step-form>
```

#### 步骤3：在页面 JS 中初始化数据

```javascript
Page({
  data: {
    steps: ['填写基本信息', '上传资料', '确认提交'],
    currentStep: 0
  },

  // 步骤变化事件处理
  onStepChange(e) {
    const { currentStep, direction } = e.detail;
    this.setData({ currentStep });
    console.log('当前步骤:', currentStep, '方向:', direction);
  }
})
```

---

### 3.3 组件属性

| 属性名 | 类型 | 默认值 | 必填 | 说明 |
|--------|------|--------|------|------|
| steps | Array | [] | 是 | 步骤标题数组，如 `['步骤1', '步骤2', '步骤3']` |
| currentStep | Number | 0 | 否 | 当前步骤索引（从 0 开始） |

---

### 3.4 组件方法

#### nextStep()

前进到下一步。只有当前步骤已完成（通过 `updateStepStatus(true)` 标记）才能前进。

**返回值：**
- `true`: 成功前进
- `false`: 已是最后一步或当前步骤未完成

**示例：**
```javascript
const stepForm = this.selectComponent('#stepForm');
const success = stepForm.nextStep();
if (!success) {
  wx.showToast({ title: '请完成当前步骤', icon: 'none' });
}
```

#### prevStep()

返回到上一步。

**返回值：**
- `true`: 成功返回
- `false`: 已是第一步

**示例：**
```javascript
const stepForm = this.selectComponent('#stepForm');
stepForm.prevStep();
```

#### goToStep(step)

跳转到指定步骤。只能跳转到已完成的步骤。

**参数：**
- `step` (Number): 目标步骤索引

**返回值：**
- `true`: 跳转成功
- `false`: 目标步骤未完成或索引无效

**示例：**
```javascript
const stepForm = this.selectComponent('#stepForm');
stepForm.goToStep(1); // 跳转到第2步
```

#### updateStepStatus(completed)

更新当前步骤的完成状态。只有标记为已完成的步骤，才能通过滑动或 `nextStep()` 前进。

**参数：**
- `completed` (Boolean): 是否完成

**示例：**
```javascript
// 在表单数据变化时调用
onInputChange(e) {
  const value = e.detail.value;
  this.setData({ inputValue: value });
  
  // 检查表单是否填写完整
  const stepForm = this.selectComponent('#stepForm');
  if (stepForm) {
    stepForm.updateStepStatus(!!value); // 有值则标记为已完成
  }
}
```

#### reset()

重置组件状态到初始状态（第一步，所有步骤标记为未完成）。

**示例：**
```javascript
const stepForm = this.selectComponent('#stepForm');
stepForm.reset();
```

#### getCurrentStepInfo()

获取当前步骤信息。

**返回值：**
```javascript
{
  currentStep: 1,           // 当前步骤索引
  maxCompletedStep: 0,      // 最大已完成步骤索引
  totalSteps: 3,            // 总步骤数
  canGoNext: true,          // 是否可以前进
  canGoPrev: true           // 是否可以后退
}
```

---

### 3.5 组件事件

#### stepchange

步骤变化时触发。

**事件对象 detail：**
```javascript
{
  currentStep: 1,      // 新的步骤索引
  direction: 'next'    // 方向：'next' 或 'prev'
}
```

**示例：**
```xml
<step-form bindstepchange="onStepChange"></step-form>
```

```javascript
onStepChange(e) {
  const { currentStep, direction } = e.detail;
  console.log('步骤变化:', currentStep, direction);
  
  // 可以在这里执行步骤切换时的逻辑
  if (direction === 'next') {
    // 前进时的逻辑
  } else {
    // 后退时的逻辑
  }
}
```

---

### 3.6 交互方式

组件支持以下三种交互方式：

1. **点击顶部步骤圆圈跳转**：点击已完成步骤的圆圈可直接跳转
2. **左右滑动切换**：在表单内容区域左右滑动（需当前步骤已完成才能向右滑）
3. **调用方法切换**：通过 `nextStep()` / `prevStep()` 方法切换

### 3.7 实际应用案例

以下是商家升级页面的完整应用示例：

**upgrade-merchant.json**
```json
{

    "step-form": "/components/step-form/step-form"
  }
}
```

**upgrade-merchant.wxml**
```xml
<step-form 
  id="merchantStepForm"
  steps="{{merchantSteps}}"
  currentStep="{{merchantCurrentStep}}"
  bindstepchange="onMerchantStepChange">
  
  <!-- 步骤1: 店铺规模 -->
  <view class="step-panel" wx:if="{{merchantCurrentStep === 0}}">
    <view class="form-item">
      <text class="label">您当前的店铺规模是？</text>
      <radio-group class="radio-group" bindchange="onShopScaleChange">
        <label class="radio-item" wx:for="{{shopScaleOptions}}" wx:key="value">
          <radio value="{{item.value}}" checked="{{merchantData.shopScale === item.value}}" />
          <text>{{item.label}}</text>
        </label>
      </radio-group>
    </view>
    
    <view class="btn-group">
      <button class="next-btn" bindtap="merchantNextStep" disabled="{{!merchantData.shopScale}}">
        下一步
      </button>
    </view>
  </view>

  <!-- 步骤2: 会员规模 -->
  <view class="step-panel" wx:if="{{merchantCurrentStep === 1}}">
    <view class="form-item">
      <text class="label">您当前拥有的会员规模是？</text>
      <radio-group class="radio-group" bindchange="onMemberScaleChange">
        <label class="radio-item" wx:for="{{memberScaleOptions}}" wx:key="value">
          <radio value="{{item.value}}" checked="{{merchantData.memberScale === item.value}}" />
          <text>{{item.label}}</text>
        </label>
      </radio-group>
    </view>
    
    <view class="btn-group">
      <button class="prev-btn" bindtap="merchantPrevStep">上一步</button>
      <button class="next-btn" bindtap="merchantNextStep" disabled="{{!merchantData.memberScale}}">
        下一步
      </button>
    </view>
  </view>

  <!-- 更多步骤... -->
</step-form>
```

**upgrade-merchant.js**
```javascript
Page({
  data: {
    // 步骤标题数组
    merchantSteps: [
      '您当前的店铺规模是？',
      '您当前拥有的会员规模是？',
      '请选择您的营业执照',
      '请输入店铺信息'
    ],
    merchantCurrentStep: 0,
    
    // 表单数据
    merchantData: {
      shopScale: 0,
      memberScale: 0,
      licenseImage: '',
      shopName: ''
    },
    
    // 选项数据
    shopScaleOptions: [
      { label: '1家店铺', value: 1 },
      { label: '2-5家店铺', value: 2 },
      { label: '6-10家店铺', value: 3 },
      { label: '10家以上店铺', value: 4 }
    ]
  },

  // 步骤变化事件
  onMerchantStepChange(e) {
    const { currentStep, direction } = e.detail;
    this.setData({ merchantCurrentStep: currentStep });
  },

  // 店铺规模选择
  onShopScaleChange(e) {
    const value = Number(e.detail.value);
    this.setData({ 'merchantData.shopScale': value });
    
    // 标记当前步骤为已完成
    const stepForm = this.selectComponent('#merchantStepForm');
    if (stepForm) {
      stepForm.updateStepStatus(true);
    }
  },

  // 会员规模选择
  onMemberScaleChange(e) {
    const value = Number(e.detail.value);
    this.setData({ 'merchantData.memberScale': value });
    
    // 标记当前步骤为已完成
    const stepForm = this.selectComponent('#merchantStepForm');
    if (stepForm) {
      stepForm.updateStepStatus(true);
    }
  },

  // 下一步
  merchantNextStep() {
    const stepForm = this.selectComponent('#merchantStepForm');
    if (stepForm) {
      const success = stepForm.nextStep();
      if (!success) {
        wx.showToast({
          title: '请完成当前步骤',
          icon: 'none'
        });
      }
    }
  },

  // 上一步
  merchantPrevStep() {
    const stepForm = this.selectComponent('#merchantStepForm');
    if (stepForm) {
      stepForm.prevStep();
    }
  }
})
```

### 3.8 使用注意事项

1. **步骤完成状态管理**：必须在每个步骤的数据变化时调用 `updateStepStatus()` 更新完成状态，否则用户无法前进到下一步。

2. **步骤内容显示**：使用 `wx:if` 根据 `currentStep` 控制每个步骤的显示，避免使用 `hidden`（会影响滑动手势）。

3. **按钮禁用状态**：建议根据表单数据的完整性动态控制"下一步"按钮的 `disabled` 状态，提供更好的用户体验。

4. **多实例使用**：同一页面可以使用多个 `step-form` 实例，通过不同的 `id` 区分即可。

## 5. 常见问题

### Q1: custom-toast 消息显示不出来？

**解决方案：**
- 检查组件是否在 WXML 中正确引用
- 检查 `id` 是否正确
- 确认 `selectComponent()` 成功获取到组件实例

### Q2: custom-picker 选择后没有触发事件？

**解决方案：**
- 确认绑定的是 `bindchange` 而不是 `bindconfirm`
- 检查 `options` 数组是否正确传入
- 如果是对象数组，确认 `range-key` 属性正确设置

### Q3: step-form 无法前进到下一步？

**解决方案：**
- 检查是否调用了 `updateStepStatus(true)` 标记当前步骤为已完成
- 确认当前步骤不是最后一步

### Q4: step-form 滑动不生效？

**解决方案：**
- 确认步骤内容使用 `wx:if` 而非 `hidden` 控制显示
- 检查当前步骤是否已标记为完成（只有完成的步骤才能向右滑）

### Q5: 如何自定义组件样式？

**解决方案：**
- custom-toast：直接修改组件的 WXSS 文件
- custom-picker：修改组件 WXSS 中的 `.picker-item-text` 等类
- step-form：在页面的 WXSS 中覆盖 `.step-panel` 等类的样式

## 4. card-display（卡片展示组件）

### 4.1 组件简介

会员卡片展示组件，用于统一渲染会员卡的背景和蒙版。支持多种背景类型（OSS图片、本地高级图片、渐变色）和蒙版图片，自动保持1.585:1的宽高比。

**已应用页面：** `/pages/card-type-manage`（卡种列表）、`/pages/card-type-update`（旧卡片展示）

---

### 4.2 快速上手

#### 步骤1：在页面 JSON 中注册组件

```json
{
  "usingComponents": {
    "card-display": "/components/card-display/card-display"
  }
}
```

#### 步骤2：在页面 WXML 中添加组件

```xml
<!-- 默认宽度（720rpx，96%屏幕宽） -->
<card-display 
  cardBgc="{{cardType.cardBgc}}" 
  cardMask="{{cardType.cardMask}}" />

<!-- 自定义宽度 -->
<card-display 
  width="{{495}}"
  cardBgc="{{cardType.cardBgc}}" 
  cardMask="{{cardType.cardMask}}" />
```

---

### 4.3 组件属性

| 属性名 | 类型 | 默认值 | 必填 | 说明 |
|--------|------|--------|------|------|
| width | Number | 720 | 否 | 卡片宽度（rpx单位），高度自动计算保持1.585:1比例 |
| cardBgc | String | '' | 否 | 背景字段，支持多种格式（见下方说明） |
| cardMask | String | '' | 否 | 蒙版字段，支持OSS图片路径或空字符串 |

---

### 4.4 cardBgc 字段格式说明

组件会根据 `cardBgc` 的第一个字符自动识别类型：

#### 1. OSS图片（以 `/` 开头）
```javascript
cardBgc: "/card/bg_20241210_abc123.png"
```
组件会自动调用 `oss.js` 获取临时签名URL并显示。

#### 2. 本地高级图片（以 `bg` 开头）
```javascript
cardBgc: "bg_black.png"
```
组件会自动拼接路径前缀 `/assets/bkgd/`，最终路径为 `/assets/bkgd/bg_black.png`。

#### 3. 渐变色（数字开头，格式：`{direct}_{seq}`）
```javascript
cardBgc: "3_1"  // 对角渐变，暗黑色方案
cardBgc: "1_5"  // 水平渐变，薄荷色方案
```

**渐变方向 (direct)**：
- `1`: 水平渐变（to right）
- `2`: 垂直渐变（to bottom）
- `3`: 对角渐变（135deg）

**渐变色方案 (seq)**：
- `1`: 暗黑（黑灰黑）
- `2`: 金色
- `3`: 日落（红黄）
- `4`: 海洋（蓝青）
- `5`: 薄荷（绿青）
- `6`: 粉橙
- `7`: 紫青
- `8`: 梦幻（青粉）
- `9`: 彩虹（粉橙紫）

#### 4. 其他情况（null、空字符串等）
显示默认蓝紫渐变背景。

---

### 4.5 cardMask 字段格式说明

#### 1. OSS图片（以 `/` 开头）
```javascript
cardMask: "/card/mask_20241210_xyz789.png"
```
组件会自动调用 `oss.js` 获取临时签名URL并显示。

#### 2. 空字符串
```javascript
cardMask: ""
```
不显示蒙版层。

---

### 4.6 常用宽度参考

| 宽度百分比 | rpx值 | 适用场景 |
|-----------|-------|---------|
| 96% | 720 | 卡片详情页、预览区（默认） |
| 92% | 690 | 列表页大卡片 |
| 66% | 495 | 两列网格布局 |
| 50% | 375 | 三列网格布局 |
| 40% | 300 | 小卡片预览 |
| 30% | 225 | 缩略图 |

**高度计算公式**：`height = width / 1.585`

### 4.8 使用注意事项

1. **宽高比保持**：组件内部自动计算高度，无需手动设置，始终保持1.585:1比例。

2. **OSS图片加载**：OSS图片需要网络请求获取临时URL，首次加载可能有延迟，组件内部已处理加载失败情况。

3. **渐变色配置**：渐变色方案与 `mask-test` 页面保持一致，确保预览效果与实际效果相同。

4. **响应式适配**：使用rpx单位，自动适配不同设备屏幕宽度。

5. **性能优化**：组件使用 `observers` 监听属性变化，只在数据变化时重新渲染，避免不必要的性能开销。

## 6. 常见问题

### Q1: custom-toast 消息显示不出来？

**解决方案：**
- 检查组件是否在 WXML 中正确引用
- 检查 `id` 是否正确
- 确认 `selectComponent()` 成功获取到组件实例

### Q2: step-form 无法前进到下一步？

**解决方案：**
- 检查是否调用了 `updateStepStatus(true)` 标记当前步骤为已完成
- 确认当前步骤不是最后一步

### Q3: step-form 滑动不生效？

**解决方案：**
- 确认步骤内容使用 `wx:if` 而非 `hidden` 控制显示
- 检查当前步骤是否已标记为完成（只有完成的步骤才能向右滑）

### Q4: card-display 显示空白或默认背景？

**解决方案：**
- 检查 `cardBgc` 格式是否正确
- 如果是OSS图片，确认路径是否正确（以 `/` 开头）
- 查看控制台是否有错误日志
- 确认OSS凭证是否有效（可在 `oss-test` 页面测试）

### Q5: card-display 图片变形？

**解决方案：**
- 组件已自动保持1.585:1比例，不会变形
- 如果背景图片本身比例不是1.585:1，会使用 `aspectFill` 模式裁剪
- 蒙版图片使用 `object-fit: fill` 完全填充，确保Canvas生成的图片正确显示

### Q6: 如何自定义组件样式？

**解决方案：**
- custom-toast：直接修改组件的 WXSS 文件
- step-form：在页面的 WXSS 中覆盖 `.step-panel` 等类的样式
- card-display：修改组件 WXSS 中的 `.card-display-container` 等类，或在页面中通过外层容器控制
