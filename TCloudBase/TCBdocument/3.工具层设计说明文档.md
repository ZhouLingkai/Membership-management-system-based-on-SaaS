# 工具层设计说明文档

> 定位：层代码（Layer），供所有云函数 `require` 调用，禁止在云函数中重复实现以下功能。
> 运行时：Node.js 18（云环境）/ Node.js 20（本地调试）
> 文件位置：`TCloudBase/layers/nodejs/`

---

## 一、层文件总览

| 文件 | 职责 |
|---|---|
| `JWTUtils.js` | JWT 签发、解析、校验 |
| `EncryptUtils.js` | AES-256-CBC 加解密、Argon2id 密码哈希与校验、UUID 生成、手机号解密 |
| `DateUtils.js` | 时间工具：当前时间、时区转换、次日零点、过期判断 |
| `ValidateUtils.js` | 通用字段校验：手机号、密码强度、预约时间段、取消规则、禁止日期 |
| `HttpUtils.js` | 云函数 HTTP 请求解析：method、headers、params 统一提取 |
| `ErrorCodes.js` | 统一错误码定义，编码规则按模块分段（1xxxx~9xxxx） |
| `ResponseUtils.js` | 统一响应包装：`success` / `fail` / `error`，调用 ErrorCodes 生成标准响应体 |

> 旧版 Java 中的 `CookieUtils`、`ReservationLockUtil` 已废弃，不迁移：
> - Cookie 管理：云函数无 HttpOnly Cookie，令牌统一由前端 Header 传递
> - 分布式锁：改用 NoSQL 唯一索引 + 原子 count 模拟乐观锁（见集合设计文档）

## 二、`JWTUtils.js`

### 依赖
```
jsonwebtoken （npm）
uuid （npm）
```

### 密钥管理
- 密钥**硬编码在层文件中**，独立开发者不上传 git，无泄露风险
- `JWT_SECRET = 'emQOFvSdF/A4GsFjCyQi3O0hLvca3t5WpWMVSsnF6h6g0b6c7SHOKQE6B4ddfJ7DPt2w6kdzVAOCL1Zhqj+6Cw=='`

### 方法说明

#### `signToken(payload, expiresIn)`
签发 JWT，自动注入 `jti`（UUID v4）和 `iat`。

| 参数 | 类型 | 说明 |
|---|---|---|
| `payload` | Object | 业务字段，见新版JWT设计文档第三章 |
| `expiresIn` | String/Number | 如 `'7d'`、`'4h'`、`'30d'` |

返回：JWT 字符串

**7种令牌签发示例**（payload字段以JWT设计文档第三章为准）：

```js
// ① 商家登录令牌（7d）
const loginToken = signToken({ uid, role: 'merchant', type: 'login', v, device_id }, '7d')

// ② 商家访问令牌（4h）
const accessToken = signToken({ uid, role: 'merchant', type: 'access', merchantId, device_id }, '4h')

// ③ 商家工作令牌（4h）
const workToken = signToken({ uid, role: 'merchant', type: 'work', merchantId, storeId, device_id }, '4h')

// ④ 员工登录令牌（2d）
const staffLoginToken = signToken({ uid, role: 'staff', type: 'login', v, device_id }, '2d')

// ⑤ 员工工作令牌（4h）
const staffWorkToken = signToken({ uid, role: 'staff', type: 'work', merchantId, storeId, permissions, device_id }, '4h')

// ⑥ 会员登录令牌（30d）
const memberLoginToken = signToken({ uid, role: 'member', type: 'login', merchantId, v, device_id }, '30d')

// ⑦ 会员业务令牌（8h）
const memberAccessToken = signToken({ uid, role: 'member', type: 'access', merchantId, device_id }, '8h')

// ⑧ 商家特权令牌（10min）
const privilegeToken = signToken({ uid, role: 'merchant', type: 'privilege', merchantId, device_id }, '10m')

// ⑨ 会员二维码令牌（5min）
const memberQrcodeToken = signToken({ uid, role: 'member', type: 'qrcode', device_id }, '5m')
```

#### `verifyToken(token)`
验证签名合法性 + 是否过期，返回解码后的 `payload`。失败抛出错误（`TokenExpiredError` / `JsonWebTokenError`）。

> 此方法只做 JWT 层校验（签名+过期），**不做**黑名单查询和 tokenVersion 校验，这两步由业务层调用。

#### `decodeToken(token)`
不验证签名，直接解码 payload。用于提取 `jti` 等字段做后续处理（如退出时加入黑名单）。

#### `getTokenRemainingSeconds(token)`
返回令牌剩余有效秒数（用于写入黑名单时设置 `expireTime`）。

### 新旧差异
| 项目 | 旧版 Java | 新版 Node.js |
|---|---|---|
| 令牌种类 | 普通/特权/工作/管理/自动登录 | ①~⑦，见JWT设计文档 |
| 签名算法 | HS256 | HS256（不变） |
| `jti` 生成 | `UUID.randomUUID().toString()` | `uuidv4()`（等价） |
| 黑名单存储 | Redis | `token_blacklist` 集合 |
| 管理令牌 | 有 | **已移除** |
| 特权令牌（二维码） | 有 | **已移除** |

## 三、`EncryptUtils.js`

### 依赖
```
crypto （Node.js 内置，无需安装）
@node-rs/argon2 （npm）
uuid （npm）
```

### 方法说明

#### `encryptAES(plainText)`
AES-256-CBC 加密，用于密码等需加密传输的字段（云函数内部使用，前端不调此函数）。

> **手机号不使用此函数存储**：手机号采用「加密传输、明文存储」策略，应调用 `decryptPhone()` 解密后直接存入数据库。

- 每次加密生成随机 16 字节 IV（随机 IV 防模式分析攻击）
- 输出格式：`Base64(IV + cipherText)`，与旧 Java 版本格式完全一致，小程序/Web 端均可对或解密
- 密钥硬编码在层文件中：`AES_SECRET_KEY = 'oUDKf7+y6SrpeJ+GCJHL31HKlRFJ1mM/1MhGB4HF/94='`（Base64 编码的 32 字节密钥，与旧版一致）

#### `decryptAES(encryptedText)`
AES-256-CBC 解密，输入格式同上。

#### `hashPassword(rawPassword)`
Argon2id 密码哈希（async）。

参数：`timeCost: 2`，`memoryCost: 65536`，`parallelism: 1`，`hashLength: 32`，`saltLength: 16`

> 与旧版 Java Argon2 参数保持一致。旧版输出 PHC 字符串（`$argon2id$...`），Node.js `argon2` 库输出格式相同，可直接验证旧哈希。

#### `verifyPassword(rawPassword, hashedPassword)`
验证密码（async），返回 Boolean。

#### `generateUUID()`
生成 UUID v4 字符串，用于 `jti`、集合 `_id` 生成等场景。

#### `decryptPhone(phoneOrCipher)`
手机号解密（智能判断）：若输入已是 11 位纯数字明文则直接返回，否则走 AES-256-CBC 解密。
前端传来的手机号统一为 AES 密文，云函数中统一调用此函数而非裸调 `decryptAES`。

> **存储规范**：解密得到 11 位明文后，直接存入数据库（明文存储）。所有涉及手机号的集合字段（`merchants.phone`、`verify_codes.phone`、`login_fail_records.identifier`）均存明文。

### 新旧差异
| 项目 | 旧版 Java | 新版 Node.js |
|---|---|---|
| AES 格式 | Base64(IV+cipher) | 相同，完全兼容 |
| 密码哈希 | Argon2id，PHC 格式 | 相同，可验证旧哈希 |
| UUID/BINARY 互转 | `uuidToBytes` / `bytesToUuid` | **已移除**，NoSQL `_id` 直接存 UUID 字符串，无需转换 |

## 四、`DateUtils.js`

### 依赖
```
无（使用 Node.js 内置 Date + Intl）
```

### 时区
全局统一：`Asia/Shanghai`（UTC+8）

### 方法说明

#### `nowDate()`
返回当前时间的 `Date` 对象（云数据库写入直接用）。

#### `nowTimestamp()`
返回当前毫秒级时间戳（`Number`）。

#### `tomorrowMidnight()`
返回次日零点的 `Date` 对象，用于 `verify_codes.expireTime`（每日清理基准时间）。

#### `addSeconds(date, seconds)`
在给定 `Date` 基础上加指定秒数，返回新 `Date`。

#### `addMinutes(date, minutes)`
同上，加分钟数。用于验证码过期时间（+5分钟）、频率限制（+1分钟）等。

#### `addDays(date, days)`
同上，加天数。用于时长会员到期时间计算。

#### `isExpired(date)`
判断给定 `Date` 是否已过期（`< nowDate()`），返回 Boolean。

#### `formatDate(date)`
返回 `'YYYY-MM-DD'` 格式字符串，用于预约记录 `reservationDate` 字段。

#### `getDayOfWeek(date)`
返回中文星期字符串（`'周一'`~`'周日'`），用于 `forbiddenDays` 匹配。

### 新旧差异
| 项目 | 旧版 Java | 新版 Node.js |
|---|---|---|
| 时区处理 | `ZoneId.of("Asia/Shanghai")` | UTC+8 手动偏移或 `Intl` |
| `LocalDateTime` 类型 | Java 类型系统 | 统一用 `Date` 对象（云数据库原生支持） |
| `toLocalDateTime` | 有 | **移除**，云数据库直接存取 `Date`，无需互转 |

## 五、`ValidateUtils.js`

### 依赖
```
无（纯 JS 逻辑）
```

### 方法说明

#### `validatePhone(phone)`
校验手机号格式（11位、1开头中国大陆号段），返回 Boolean。

#### `validatePassword(password)`
密码强度校验：8~20位，必须包含字母和数字，返回 `{ valid: Boolean, message: String }`。

#### `validateTimeSlots(timeSlots)`
校验预约时间段列表（对应 `reservation_templates.reservationTimeList`）：
- 格式：`['HH:mm-HH:mm', ...]`
- 规则：格式合法、开始 < 结束、时长 ≥ 30 分钟、时间段不交叉、从小到大排序
- 失败抛出 `Error`（含错误描述）

#### `validateCancelRules(cancelRules)`
校验取消规则格式（对应 `reservation_templates.cancelRule`）：
- 格式：对象数组 `[{ minBefore: Number, value: Number|null, penaltyType: String }, ...]`
- `penaltyType` 枚举：`'prohibit'`（禁止取消，value=null）| `'ratio'`（按比例，0<value≤1）| `'fixed'`（固定金额，value≥0）
- 失败抛出 `Error`（含错误描述）

#### `calculateCancelFee(minutesBeforeStart, cancelRules, totalAmount)`
计算取消违约费：
- 匹配逻辑：取 `minBefore >= minutesBeforeStart` 中最小的规则；已开始（<0）取 `minBefore` 最小规则
- 返回 `{ fee: Number|null, prohibited: Boolean, ruleDesc: String }`
  - `prohibited=true`：该时间段禁止取消，`fee=null`；业务层根据此字段抛出 `RESERVATION_CANCEL_DENIED`
  - `prohibited=false`：`fee` 为应扣金额（可为 0）

#### `validateForbiddenDays(forbiddenDays)`
校验禁止日期列表格式：
- 支持中文星期（`'周一'`~`'周日'`）和具体日期（`'YYYY-MM-DD'`）
- 过期的具体日期自动过滤（不报错）

#### `isDateReservable(dateStr, templateForbiddenDays, resourceForbiddenDays, customizeForbidden)`
判断某日期是否可预约：
- `dateStr`：`'YYYY-MM-DD'` 字符串
- `customizeForbidden`：`0` = 仅用模板规则，`1` = 模板+资源都需通过
- 支持"负负得正"逻辑：星期 + 具体日期同时命中 → 恢复可预约

#### `validateReservationContinuousTime(minTime, maxTime)`
校验连续预约时间范围：`0 ≤ minTime ≤ maxTime ≤ 1440`（分钟）。

### 新旧差异
| 项目 | 旧版 Java | 新版 Node.js |
|---|---|---|
| Spring Validation 注解 | 有（`@NotNull` 等） | 无，改为函数式校验，云函数入口显式调用 |
| `ResourceValidator` 次数卡校验 | 有（cardType=2 时 min=max） | **移除**，新版会员类型无次数卡 |
| `BusinessException` 抛出 | Spring 异常体系 | 抛出普通 `Error`，云函数统一 catch 处理 |

## 六、使用规范

1. **云函数引入方式**（层已挂载后直接 require）：
   ```js
   const { signToken, verifyToken } = require('/opt/nodejs/JWTUtils')
   const { encryptAES, hashPassword, decryptPhone } = require('/opt/nodejs/EncryptUtils')
   const { nowDate, tomorrowMidnight } = require('/opt/nodejs/DateUtils')
   const { validatePhone, isDateReservable } = require('/opt/nodejs/ValidateUtils')
   const { parseRequest } = require('/opt/nodejs/HttpUtils')
   const ErrorCodes = require('/opt/nodejs/ErrorCodes')
   const { success, fail, error } = require('/opt/nodejs/ResponseUtils')
   ```

2. **错误处理原则**：工具层只抛出 `Error`，云函数统一 `try/catch` 捕获后调用 `error(e)` 返回 99999，不在工具层处理响应逻辑。

3. **云函数标准模板**（三层鉴权按需取用，详见JWT设计文档第七章）：
   ```js
   const { parseRequest } = require('/opt/nodejs/HttpUtils')
   const { verifyToken } = require('/opt/nodejs/JWTUtils')
   const { success, fail, error } = require('/opt/nodejs/ResponseUtils')
   const ErrorCodes = require('/opt/nodejs/ErrorCodes')

   exports.main = async (event, context) => {
     try {
       const { params, headers } = parseRequest(event)

       // ── 层① 无状态校验（所有鉴权接口必须）─────────────────────
       const token = (headers['authorization'] || '').replace('Bearer ', '')
       if (!token) return fail(ErrorCodes.TOKEN_MISSING)
       let payload
       try {
         payload = verifyToken(token)
       } catch (e) {
         return fail(e.name === 'TokenExpiredError' ? ErrorCodes.TOKEN_EXPIRED : ErrorCodes.TOKEN_MISSING)
       }

       // ── 层② 黑名单校验（写操作 / login / work 令牌时加）────────
       // const { total } = await db.collection('token_blacklist').where({ jti: payload.jti }).count()
       // if (total > 0) return fail(ErrorCodes.TOKEN_BLACKLISTED)

       // ── 层③ 版本号校验（资金/权限写操作，含 v 字段的令牌时加）──
       // const doc = await db.collection('merchants').doc(payload.uid).get()
       // if (!doc.data || doc.data.tokenVersion !== payload.v) return fail(ErrorCodes.TOKEN_VERSION_MISMATCH)

       // 业务逻辑...
       return success({ ... }, '操作成功')
     } catch (e) {
       return error(e)
     }
   }
   ```

## 八、`ErrorCodes.js`（统一错误码）

### 依赖
```
无（纯 JS 常量）
```

### 编码规则

| 段 | 模块 |
|---|---|
| `1xxxx` | 令牌相关 |
| `2xxxx` | 验证码相关 |
| `3xxxx` | 商家账号 |
| `4xxxx` | 店铺 |
| `5xxxx` | 员工 |
| `6xxxx` | 会员 |
| `7xxxx` | 会员卡/资产 |
| `8xxxx` | 预约 |
| `9xxxx` | 通用参数/系统（`99999` = 系统异常） |

### 使用方式
```js
const ErrorCodes = require('ErrorCodes')
return fail(ErrorCodes.MERCHANT_NOT_FOUND)
// → { code: 30002, data: {}, message: '商家账号不存在' }
```

## 九、`ResponseUtils.js`（统一响应包装）

### 依赖
```
ErrorCodes（层内依赖）
```

### 方法说明

#### `success(data, message)`
成功响应，`code=200`。`data` 默认 `{}`，`message` 默认 `'操作成功'`。

#### `fail(errorItem, extra)`
业务失败响应，接收 `ErrorCodes` 中的错误项。`extra` 可传附加数据（如锁定截止时间、剩余秒数）。

#### `error(err)`
系统异常兜底，`code=99999`。自动将 err 打印到云函数日志，不暴露给前端。

## 七、`HttpUtils.js`（云函数 HTTP 请求解析）

> 腾讯云开发云函数通过 `event` 对象接收 HTTP 触发器请求，无 Express req/res。

### 依赖
```
无（纯 JS 逻辑）
```

### 方法说明

#### `parseRequest(event)`
解析云函数 `event`，统一返回请求三要素。

返回：
```js
{
  method,   // String：'GET' / 'POST' 等
  headers,  // Object：全部请求头
  params    // Object：业务参数（GET 来自 queryString，POST 来自 body JSON）
}
```

示例：
```js
const { method, headers, params } = parseRequest(event)
const token = headers['authorization']  // 取令牌
const { storeId } = params              // 取业务参数
```

内部实现逻辑（供参考，不在云函数中重复写）：
```js
const method = event.httpMethod || 'UNKNOWN'
const headers = event.headers || {}
let params = {}
if (method === 'GET') {
  params = event.queryString || {}
} else if (method === 'POST') {
  params = event.body ? JSON.parse(event.body) : {}
}
```
