# 模块1：令牌管理模块

> 云函数路径：`/api/v1/tokens/`
> 所有接口均需先调用 `HttpUtils.parseRequest(event)` 解析入参。
> 响应统一格式：`{ code, data, message }`，`code=200` 为成功，其余为失败。

---

## 接口列表

| # | 接口名称 | 云函数路径建议 | HTTP 方法 | 功能 |
|---|---|---|---|---|
| 1 | 获取商家工作令牌 | `/api/v1/tokens/work` | POST | 商家用访问令牌②换工作令牌③ |
| 2 | 令牌刷新 | `/api/v1/tokens/refresh` | POST | access/work令牌写黑名单后签发同类新令牌 |
| 3 | 令牌注销（退出登录） | `/api/v1/tokens/logout` | POST | 批量将若干令牌 jti 写入黑名单，通用退出接口 |
| 4 | 获取商家特权令牌 | `/api/v1/tokens/privilege` | POST | 商家用访问令牌②+二级密码换特权令牌⑧（10min） |
| 5 | 获取会员二维码令牌 | `/api/v1/tokens/memberQrcode` | POST | 会员用业务令牌⑦换二维码令牌⑨（5min，单次有效） |

> 云函数 HTTP 触发器不支持 DELETE/PUT 路径参数，改用 POST + body 传 jti。

## 集合依赖

| 集合 | 用途 |
|---|---|
| `merchants` | 查询商家信息，校验 merchantId 归属 |
| `stores` | 校验 storeId 归属该商家 |
| `token_blacklist` | jti 黑名单读写 |

### `token_blacklist` 集合结构

```js
{
  _id: String,
  jti: String,          // JWT jti 字段，令牌唯一标识（唯一索引）
  expireTime: Date      // 与原令牌 exp 一致，过期后文档无意义
}
```

**索引：** `jti`（唯一索引），`expireTime`（定时清理，每天凌晨）

## 接口1：获取商家工作令牌

- **URL**：`/api/v1/tokens/work`
- **HTTP Method**：POST
- **认证**：需携带商家访问令牌（②，type=`access`，role=`merchant`）

### 请求头

| 字段 | 必填 | 说明 |
|---|---|---|
| `Authorization` | 是 | `Bearer <商家访问令牌②>` |
| `Content-Type` | 是 | `application/json` |

### 请求参数（body）

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `storeId` | String | 是 | 目标工作店铺 `stores._id` |

### 核心逻辑

1. `verifyToken(token)` 验证访问令牌②签名与过期
2. 校验 `payload.role === 'merchant'` && `payload.type === 'access'`
3. 查询 `stores` 集合，校验 `storeId` 存在且 `merchantId === payload.merchantId`
4. 用原 `uid`、`merchantId`、`device_id` + 新 `storeId` 签发商家工作令牌③（4h）
5. 返回新令牌

### 响应结果

| 字段 | 类型 | 说明 |
|---|---|---|
| `data.workToken` | String | `Bearer <商家工作令牌③>`（4h） |
| `data.workTokenExpireTime` | Number | 工作令牌过期 Unix 毫秒时间戳（4h后） |
| `data.storeId` | String | 绑定的店铺 ID |
| `data.merchantId` | String | 商家 ID |

### 响应示例

```json
{
  "code": 200,
  "data": {
    "workToken": "Bearer eyJhbGci...",
    "storeId": "uuid-store-xxx",
    "merchantId": "uuid-merchant-xxx"
  },
  "message": "工作令牌签发成功"
}
```

### 错误码

| code | ErrorCodes 键 | 场景 |
|---|---|---|
| 10001 | `TOKEN_MISSING` | Authorization 头为空或令牌格式非法 |
| 10002 | `TOKEN_EXPIRED` | 令牌已过期 |
| 10003 | `TOKEN_BLACKLISTED` | 令牌已注销（在黑名单中） |
| 10004 | `TOKEN_INSUFFICIENT_ROLE` | 令牌角色不符（非 merchant） |
| 10005 | `TOKEN_TYPE_MISMATCH` | 令牌类型不符（非 access） |
| 40001 | `STORE_NOT_FOUND` | storeId 不存在 |
| 40002 | `STORE_NOT_BELONG` | storeId 不归属该商家 |

## 接口2：令牌刷新

- **URL**：`/api/v1/tokens/refresh`
- **HTTP Method**：POST
- **认证**：携带待刷新令牌
- **支持刷新的令牌类型**：② `access`、③ `work`（login 令牌不可刷新）
- **严格使用场景**：仅当令牌剩余有效期 **≤10 分钟**时才允许刷新；前端负责检测时机，后端强制校验，有效期过长时拒绝

### 请求头

| 字段 | 必填 | 说明 |
|---|---|---|
| `Authorization` | 是 | `Bearer <待刷新令牌>` |
| `Content-Type` | 是 | `application/json` |

### 请求参数（body）

无（所有信息从令牌 payload 提取）

### 核心逻辑

1. `verifyToken(token)` 验证签名与过期（令牌已过期则拒绝，不允许刷新已过期令牌）
2. `decodeToken(token)` 取 payload
3. 校验 `payload.type` 为 `access` 或 `work`，若为 `login` 则拒绝
4. `getTokenRemainingSeconds(token)` 计算剩余秒数，若 **> 600 秒（10分钟）则拒绝**，返回剩余秒数供前端参考
5. 将旧令牌 `jti` 写入 `token_blacklist`（`expireTime` 取原令牌 `exp` 对应 Date）
6. 从 payload 取业务字段，按原令牌类型重新签发同类型新令牌（全新过期时间）：
   - `access` → 重签 access（4h），字段：`uid, role, type, merchantId, device_id`
   - `work`（商家）→ 重签 work（4h），字段：`uid, role, type, merchantId, storeId, device_id`
   - `work`（员工）→ 重签 work（4h），字段：`uid, role, type, merchantId, storeId, permissions, device_id`
7. 返回新令牌、新过期时间、旧 jti

### 响应结果

| 字段 | 类型 | 说明 |
|---|---|---|
| `data.newToken` | String | `Bearer <新令牌>` |
| `data.newExpireTime` | Number | 新令牌过期 Unix 毫秒时间戳（4h后） |
| `data.storeId` | String | ① work 令牌刷新时返回，绑定店铺 ID |
| `data.oldJti` | String | 已写入黑名单的旧令牌 jti |

### 响应示例

```json
{
  "code": 200,
  "data": {
    "newToken": "Bearer eyJhbGci...",
    "newExpireTime": 1744682400000,
    "storeId": "uuid-store-xxx",
    "oldJti": "uuid-old-jti-xxx"
  },
  "message": "令牌刷新成功"
}
```

### 错误码

| code | ErrorCodes 键 | 场景 |
|---|---|---|
| 10001 | `TOKEN_MISSING` | Authorization 头为空或令牌格式非法 |
| 10002 | `TOKEN_EXPIRED` | 令牌已过期（过期令牌不允许刷新） |
| 10007 | `TOKEN_LOGIN_NO_REFRESH` | login 令牌不支持刷新 |
| 10005 | `TOKEN_TYPE_MISMATCH` | privilege / qrcode 等令牌类型不支持刷新 |
| 10006 | `TOKEN_REFRESH_TOO_EARLY` | 剩余有效期 > 10 分钟，暂不允许刷新，附带 `{ remainSeconds: N }` |

## 接口3：令牌注销（退出登录）

- **URL**：`/api/v1/tokens/logout`
- **HTTP Method**：POST
- **认证**：需携带本次操作的主令牌（用于身份确认，通常为访问令牌②），同时 body 传入所有待注销令牌
- **通用性**：商家、员工、会员退出均可调用此接口

### 请求头

| 字段 | 必填 | 说明 |
|---|---|---|
| `Authorization` | 是 | `Bearer <主令牌>` （access 或 work 均可，用于身份校验） |
| `Content-Type` | 是 | `application/json` |

### 请求参数（body）

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `tokens` | Array\<String\> | 是 | 待注销令牌字符串列表（JWT 原文，不含 `Bearer ` 前缀），后端自行 `decodeToken` 提取 jti 和 exp |

> `tokens` 数组中可包含任意类型令牌（login①、access②、work③），至少传 1 个。

### 请求示例

```json
{
  "tokens": [
    "eyJhbGci...loginToken",
    "eyJhbGci...accessToken",
    "eyJhbGci...workToken"
  ]
}
```

### 核心逻辑

1. `verifyToken(mainToken)` 验证 Header 中主令牌签名与过期（主令牌本身合法才允许注销操作）
2. 校验 `tokens` 为非空数组，长度 ≤ 10（防滥用）
3. 遍历 `tokens`：
   - `decodeToken(t)` 提取 `jti`、`exp`（不校验签名，已过期令牌也允许注销）
   - 忽略 `jti` 已在黑名单的条目（幂等）
   - 批量写入 `token_blacklist`：`{ jti, expireTime: new Date(exp * 1000) }`
4. 返回成功，附带实际写入条数

> **幂等设计**：重复调用不报错，已在黑名单的 jti 直接跳过。

### 响应结果

| 字段 | 类型 | 说明 |
|---|---|---|
| `data.revokedCount` | Number | 本次实际写入黑名单的令牌数（已存在的不计入） |

### 响应示例

```json
{
  "code": 200,
  "data": { "revokedCount": 3 },
  "message": "退出登录成功"
}
```

### 错误码

| code | ErrorCodes 键 | 场景 |
|---|---|---|
| 10001 | `TOKEN_MISSING` | Header 主令牌为空 |
| 10002 | `TOKEN_EXPIRED` | Header 主令牌已过期 |
| 90002 | `PARAM_MISSING` | `tokens` 为空或非数组 |
| 90001 | `PARAM_INVALID` | `tokens` 超过上限（> 10） |

## 接口4：获取商家特权令牌

- **URL**：`/api/v1/tokens/privilege`
- **HTTP Method**：POST
- **认证**：需携带商家访问令牌（②，type=`access`，role=`merchant`）

### 请求头

| 字段 | 必填 | 说明 |
|---|---|---|
| `Authorization` | 是 | `Bearer <商家访问令牌②>` |
| `Content-Type` | 是 | `application/json` |

### 请求参数（body）

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `sndPswd` | String | 是 | 二级密码（AES 密文，云函数内 `decryptAES` 解密后 Argon2 验证） |

### 核心逻辑

1. `verifyToken(token)` 验证访问令牌②签名与过期
2. 校验 `payload.role === 'merchant'` && `payload.type === 'access'`
3. 查询 `token_blacklist`，jti 在黑名单则拒绝
4. `decryptAES(sndPswd)` 解密二级密码
5. 查询 `merchants`（`_id === payload.uid`）
6. `argon2.verify(merchant.sndPswd, rawSndPswd)` 校验二级密码，失败则拒绝
7. 用原 `uid`、`merchantId`、`device_id` 签发商家特权令牌⑧（10min）
8. 返回特权令牌

### 响应结果

| 字段 | 类型 | 说明 |
|---|---|---|
| `data.privilegeToken` | String | `Bearer <商家特权令牌⑧>`（10min） |
| `data.privilegeTokenExpireTime` | Number | 特权令牌过期 Unix 毫秒时间戳（10min后） |

### 响应示例

```json
{
  "code": 200,
  "data": {
    "privilegeToken": "Bearer eyJhbGci...",
    "privilegeTokenExpireTime": 1744683000000
  },
  "message": "特权令牌签发成功"
}
```

### 错误码

| code | ErrorCodes 键 | 场景 |
|---|---|---|
| 10001 | `TOKEN_MISSING` | Authorization 头为空或令牌格式非法 |
| 10002 | `TOKEN_EXPIRED` | 访问令牌已过期 |
| 10003 | `TOKEN_BLACKLISTED` | 访问令牌已注销 |
| 10004 | `TOKEN_INSUFFICIENT_ROLE` | 令牌角色不符（非 merchant） |
| 10005 | `TOKEN_TYPE_MISMATCH` | 令牌类型不符（非 access） |
| 90002 | `PARAM_MISSING` | 缺少 sndPswd 参数 |
| 90001 | `PARAM_INVALID` | sndPswd 解密失败 |
| 30002 | `MERCHANT_NOT_FOUND` | 商家不存在 |
| 30005 | `MERCHANT_SND_PSWD_WRONG` | 二级密码错误 |

- **认证**：需携带会员业务令牌（⑦，type=`access`，role=`member`）

### 请求头

| 字段 | 必填 | 说明 |
|---|---|---|

| `Content-Type` | 是 | `application/json` |

### 请求参数（body）

无

## 通用说明

### jti 黑名单校验（业务层责任）

每个需要鉴权的接口，在 `verifyToken` 之后需自行查询 `token_blacklist`：

```js
const blacklisted = await db.collection('token_blacklist')
  .where({ jti: payload.jti }).count();
if (blacklisted.total > 0) return fail(ErrorCodes.TOKEN_BLACKLISTED);
```

> 根据 JWT 设计文档第四章：access 令牌依赖短有效期，可视情况跳过黑名单查询以降低 DB 压力。

### 令牌 Payload 字段速查

| 令牌 | 关键字段 |
|---|---|
| ② 商家访问令牌 | `uid, role='merchant', type='access', merchantId, device_id` |
| ③ 商家工作令牌 | `uid, role='merchant', type='work', merchantId, storeId, device_id` |
| ⑧ 商家特权令牌 | `uid, role='merchant', type='privilege', merchantId, device_id` |
| ⑦ 会员业务令牌 | `uid, role='member', type='access', merchantId, device_id` |
| ⑨ 会员二维码令牌 | `uid, role='member', type='qrcode', device_id` |
