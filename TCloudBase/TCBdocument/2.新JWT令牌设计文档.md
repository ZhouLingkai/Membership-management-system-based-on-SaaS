# 新版JWT令牌设计文档

> 适用系统：三层解耦式会员管理系统（腾讯云开发 CloudBase）
> 令牌库：`jsonwebtoken`（Node.js）
> 签名算法：HS256

---

## 一、令牌总览

| # | 令牌名称 | 使用实体 | 有效期 | 用途 |
|---|---|---|---|---|
| ① | 商家登录令牌 | 商家 | 7d | 维持登录状态，换取②；自动登录入口 |
| ② | 商家访问令牌 | 商家 | 4h | 执行商家全局操作（跨店查询、账户管理等） |
| ③ | 商家工作令牌 | 商家 | 4h | 执行单店铺内操作（会员办理、充值、消费等） |
| ④ | 员工登录令牌 | 员工 | 2d | 维持登录状态，换取⑤；自动登录入口 |
| ⑤ | 员工工作令牌 | 员工 | 4h | 执行单店铺内非删除操作 |
| ⑥ | 会员登录令牌 | 会员 | 90d | 维持登录状态，换取⑦；自动登录入口 |
| ⑦ | 会员业务令牌 | 会员 | 8h | 执行会员业务接口（查询、预约等） |
| ⑧ | 商家特权令牌 | 商家 | 10min | 执行危险操作（删除等），需二级密码换取 |
| ⑨ | 会员二维码令牌 | 会员 | 5min | 扫码验证身份，单次有效（使用后立即写黑名单） |

## 二、令牌获取流程

### 2.1 商家端

```
账密登录/注册时
  └─→ 返回：① 商家登录令牌（7d）+ ② 商家访问令牌（4h）

自动登录（携带①）
  └─→ 返回：② 商家访问令牌（4h）

换取工作令牌（携带②，指定 storeId）
  └─→ 返回：③ 商家工作令牌（4h）

获取特权令牌（携带②，输入二级密码）
  └─→ 返回：⑧ 商家特权令牌（10min）
```

### 2.2 员工端

```
账密登录（账号名@邀请码 + 密码）
  └─→ 返回：④ 员工登录令牌（2d）+ ⑤ 员工工作令牌（4h）
      （员工一账号绑定一店铺，登录即自动换取工作令牌）

自动登录（携带④）
  └─→ 返回：⑤ 员工工作令牌（4h）

④ 到期（2天）→ 需重新账密登录
```

### 2.3 会员端

```
微信手机号一键登录（merchantId + 微信手机号）
  └→ 返回：⑥ 会员登录令牌（90d）+ ⑦ 会员业务令牌（8h）

自动登录（携带⑥）
  └─→ 返回：⑦ 会员业务令牌（8h）

获取二维码（携带⑦）
  └─→ 返回：⑨ 会员二维码令牌（5min，单次有效）
```

## 三、Payload 设计

### 3.1 公共字段（所有令牌均包含）

| 字段 | 类型 | 说明 |
|---|---|---|
| `uid` | String | 用户唯一ID（对应集合 `_id`） |
| `role` | String | 角色：`merchant` / `staff` / `member` |
| `type` | String | 令牌类型：`login` / `access` / `work` |
| `jti` | String | 令牌唯一ID（UUID v4），用于黑名单管理 |
| `device_id` | String | 设备ID，防止跨设备调用 |
| `exp` | Number | 过期时间（JWT标准字段，Unix时间戳，秒级） |
| `iat` | Number | 签发时间（JWT标准字段） |

### 3.2 ① 商家登录令牌

```js
{
  uid: String,          // merchants._id
  role: 'merchant',
  type: 'login',
  v: Number,            // tokenVersion，全局失效校验（改密/全设备退出时自增）
  jti: String,
  device_id: String,
  exp, iat
}
```

### 3.3 ② 商家访问令牌

```js
{
  uid: String,          // merchants._id
  role: 'merchant',
  type: 'access',
  merchantId: String,   // merchants._id（冗余，便于业务层直接使用）
  jti: String,
  device_id: String,
  exp, iat
}
```

> 不含 `v`，访问令牌短期有效（4h），黑名单机制足够覆盖失效场景。

### 3.4 ③ 商家工作令牌

```js
{
  uid: String,          // merchants._id
  role: 'merchant',
  type: 'work',
  merchantId: String,
  storeId: String,      // 操作限定的店铺 stores._id
  jti: String,
  device_id: String,
  exp, iat
}
```

> 商家拥有全部权限，工作令牌不含 `permissions` 字段。

### 3.5 ④ 员工登录令牌

```js
{
  uid: String,          // staffs._id
  role: 'staff',
  type: 'login',
  v: Number,            // tokenVersion（改密/重置时自增）
  jti: String,
  device_id: String,
  exp, iat
}
```

### 3.6 ⑤ 员工工作令牌

```js
{
  uid: String,          // staffs._id
  role: 'staff',
  type: 'work',
  merchantId: String,   // 所属商家 merchants._id
  storeId: String,      // 绑定店铺 stores._id
  permissions: Array,   // 操作权限列表，如 ['recharge', 'consume', 'member_create']
  jti: String,
  device_id: String,
  exp, iat
}
```

> 员工无全局删除权限，`permissions` 中不含删除类操作。

### 3.7 ⑥ 会员登录令牌

```js
{
  uid: String,          // members._id
  role: 'member',
  type: 'login',
  merchantId: String,   // 所属商家（会员登录凭证之一，限定查询范围）
  v: Number,            // tokenVersion（改密/注销时自增）
  jti: String,
  device_id: String,
  exp, iat
}
```

### 3.8 ⑦ 会员业务令牌

```js
{
  uid: String,          // members._id
  role: 'member',
  type: 'access',
  merchantId: String,   // 限定该会员只能访问本商家下的数据
  jti: String,
  device_id: String,
  exp, iat
}
```

### 3.9 ⑧ 商家特权令牌

```js
{
  uid: String,          // merchants._id
  role: 'merchant',
  type: 'privilege',
  merchantId: String,
  jti: String,
  device_id: String,
  exp,                  // iat + 10分钟
  iat
}
```

> 需商家访问令牌② + 二级密码换取。业务层必须校验层①+层②（黑名单）+类型为 `privilege`。证明不单次消耗，10分钟内可重复用于多次危险操作。

### 3.10 ⑨ 会员二维码令牌

```js
{
  uid: String,          // members._id
  role: 'member',
  type: 'qrcode',
  jti: String,
  device_id: String,
  exp,                  // iat + 5分钟
  iat
}
```

> 单次有效：业务层校验通过后立即将 `jti` 写入 `token_blacklist`，不含 `merchantId`，由扫码方业务接口自行确认会员归属。

## 四、令牌失效机制

### 4.1 两种失效策略

| 策略 | 触发场景 | 实现方式 |
|---|---|---|
| **全局失效**（`tokenVersion` 自增） | 改密、全设备退出 | 登录令牌内含 `v`，校验时比对集合中最新 `tokenVersion` |
| **单令牌失效**（黑名单） | 单设备退出、令牌刷新、令牌注销 | 将 `jti` 写入 `token_blacklist` 集合 |

> `v` 字段**仅登录令牌携带**。访问令牌、工作令牌、业务令牌均不含 `v`，依赖黑名单或短有效期自然失效。

### 4.2 三层校验规范（业务层，非JWT工具层）

> `JWTUtils.verifyToken()` 只做层①，②③由云函数业务层按需调用。

#### 层① 无状态签名校验（所有鉴权接口必须）

```js
const payload = verifyToken(token)
// 签名非法 → JsonWebTokenError
// 令牌过期 → TokenExpiredError
// 以上均进入 catch，返回 10001/10002
```

适用：**除注册、发验证码外的全部接口**，无需查库，纯本地运算。

#### 层② jti 黑名单校验（查 `token_blacklist` 集合）

管控场景：主动退出登录、令牌刷新时旧令牌失效。

| 令牌 | 是否查黑名单 | 说明 |
|---|---|---|
| ① 登录令牌 | **必须** | 退出登录后黑名单是唯一失效手段 |
| ③⑤ 工作令牌 | **必须** | 工作令牌刷新后旧令牌写黑名单 |
| ②⑦ 访问/业务令牌 | **可选**（写操作建议查） | 短有效期（4h/8h），退出时写黑名单，读操作可跳过降低DB压力 |

```js
const { total } = await db.collection('token_blacklist').where({ jti: payload.jti }).count()
if (total > 0) return fail(ErrorCodes.TOKEN_BLACKLISTED)
```

#### 层③ tokenVersion 版本校验（查对应角色集合）

管控场景：改密码（全局失效所有登录令牌）、员工权限变更。

| 令牌 | 是否校验版本 | 说明 |
|---|---|---|
| ① 登录令牌（含 `v`） | **写操作必须**，读可跳过 | 与 `merchants/staffs/members.tokenVersion` 比对 |
| ③⑤ 工作令牌 | **写操作必须**，读可跳过 | 与 `staffs.tokenVersion` 比对（员工权限变更后自增） |
| ②⑦ 访问/业务令牌 | 不含 `v`，**跳过** | 无版本字段，短有效期自然失效 |

```js
// 商家：通过登录令牌换取访问令牌时，在自动登录接口校验
const merchant = await db.collection('merchants').doc(payload.uid).get()
if (merchant.data.tokenVersion !== payload.v) return fail(ErrorCodes.TOKEN_VERSION_MISMATCH)

// 员工工作令牌：写操作时校验
const staff = await db.collection('staffs').doc(payload.uid).get()
if (staff.data.tokenVersion !== payload.v) return fail(ErrorCodes.TOKEN_VERSION_MISMATCH)
```

> **可选剥离原则**：读接口跳过②③可降低数据库压力，加快响应。敏感写操作（充值/退款/改密）必须执行②③。

### 4.3 退出登录行为

| 场景 | 行为 |
|---|---|
| 商家单设备退出 | 将①②③的 `jti` 写入黑名单 |
| 商家全设备退出 / 改密 | `tokenVersion` 自增（使所有①失效），同时将当前设备②③写入黑名单 |
| 员工主动退出 | 将④⑤的 `jti` 写入黑名单 |
| 会员主动退出 | 将⑥⑦的 `jti` 写入黑名单 |
| 会员二维码扫码使用 | 业务层处理完毕后，将⑨的 `jti` 写入黑名单（单次有效） |

## 五、令牌刷新机制

| 令牌 | 刷新策略 |
|---|---|
| ② 商家访问令牌（4h） | 临近过期（≤10分钟）时，携带①自动登录接口换取新② |
| ③ 商家工作令牌（4h） | 临近过期时，携带②重新换取③；旧③写入黑名单 |
| ⑤ 员工工作令牌（4h） | 临近过期时，携带④自动登录接口换取新⑤；旧⑤写入黑名单 |
| ⑦ 会员业务令牌（8h） | 临近过期时，携带⑥自动登录接口换取新⑦ |
| ⑧ 商家特权令牌（10min） | **不支持刷新**，过期后需重新输入二级密码调用 `token/privilege` 接口获取 |
| ⑨ 会员二维码令牌（5min） | **不支持刷新**，过期后需重新调用 `token/memberQrcode` 接口获取 |
| ① ④ ⑥ 登录令牌 | 到期后需重新登录（商家/员工账密，会员微信手机号） |

## 七、云函数鉴权调用规范（开发速查）

> 以下为云函数开发时的标准调用决策树，避免遗漏或过度查库。

### 7.1 接口分类与对应校验层

| 接口类型 | 示例 | 层① | 层② | 层③ |
|---|---|---|---|---|
| 无需鉴权 | 注册、发验证码 | ✗ | ✗ | ✗ |
| 读操作（低敏感） | 查商家信息、查会员列表 | ✓ | ✗ | ✗ |
| 读操作（高敏感） | 查交易记录、查积分 | ✓ | ✓（可选） | ✗ |
| 写操作（普通） | 修改商家资料、修改会员名 | ✓ | ✓ | ✗ |
| 写操作（资金/权限） | 充值、消费、退款、改密、改权限 | ✓ | ✓ | ✓ |
| 自动登录接口 | 携带①④⑥换取短令牌 | ✓ | ✓ | ✓ |
| 令牌刷新接口 | 携带②③⑤⑦换取新令牌 | ✓ | ✓ | ✗ |

### 7.2 令牌类型与可调用校验层

| 令牌 | 层①可用 | 层②可用 | 层③可用 | `v` 字段 |
|---|---|---|---|---|
| ① 商家登录令牌 | ✓ | ✓ | ✓ | ✓ |
| ② 商家访问令牌 | ✓ | ✓（写建议查） | ✗ | ✗ |
| ③ 商家工作令牌 | ✓ | ✓ | ✓（staffs无此令牌，商家工作令牌跳过③） | ✗ |
| ④ 员工登录令牌 | ✓ | ✓ | ✓ | ✓ |
| ⑤ 员工工作令牌 | ✓ | ✓ | ✓ | ✗（`v` 在④中，⑤不含） |
| ⑥ 会员登录令牌 | ✓ | ✓ | ✓ | ✓ |
| ⑦ 会员业务令牌 | ✓ | ✓（写建议查） | ✗ | ✗ |
| ⑧ 商家特权令牌 | ✓ | ✓（**必须**） | ✗ | ✗ |
| ⑨ 会员二维码令牌 | ✓ | ✓（**必须**，单次有效核心） | ✗ | ✗ |

> **商家工作令牌③** 不含 `v`，层③跳过；员工工作令牌⑤的版本号在登录令牌④中，⑤本身不携带，但员工的 `staffs.tokenVersion` 会在权限变更时自增，此时通过自动登录接口（携带④）来感知失效，不在⑤上直接校验。

### 7.3 标准云函数鉴权代码片段

```js
// ── 层① 无状态校验（所有鉴权接口）──────────────────────────
const token = (headers['authorization'] || '').replace('Bearer ', '')
if (!token) return fail(ErrorCodes.TOKEN_MISSING)
let payload
try {
  payload = verifyToken(token)
} catch (e) {
  return fail(e.name === 'TokenExpiredError' ? ErrorCodes.TOKEN_EXPIRED : ErrorCodes.TOKEN_MISSING)
}

// ── 层② 黑名单校验（写操作 / login / work 令牌）───────────────
const { total } = await db.collection('token_blacklist').where({ jti: payload.jti }).count()
if (total > 0) return fail(ErrorCodes.TOKEN_BLACKLISTED)

// ── 层③ 版本号校验（资金/权限写操作，含 v 字段的令牌）──────────
const userDoc = await db.collection('merchants').doc(payload.uid).get()  // 或 staffs / members
if (!userDoc.data || userDoc.data.tokenVersion !== payload.v) {
  return fail(ErrorCodes.TOKEN_VERSION_MISMATCH)
}
```

## 六、与旧版 Java 系统的主要差异

| 维度 | 旧版（Java + Redis） | 新版（Node.js + CloudBase） |
|---|---|---|
| 令牌存储 | Redis 存 jti / tokenVersion 缓存 | 不缓存，tokenVersion 直接读集合字段 |
| 黑名单 | Redis `blacklist:{jti}` | `token_blacklist` NoSQL 集合 |
| 自动登录轮换 | 使用3次后轮换新令牌 | 依赖有效期自然管理，无轮换计数 |
| 管理令牌 | 独立管理令牌（二级密码换取） | 新增⑧商家特权令牌，二级密码换取，10分钟有效 |
| 特权令牌（二维码） | 单次有效，扫码办卡/充值/消费 | **已移除**，新版无扫码场景 |
| 会员令牌 | 无独立会员令牌体系 | 新增⑥⑦，会员端独立令牌体系 |
| 版本号校验 | Spring Security 拦截器强制全量校验 | 业务层可选，读操作可跳过 |
